DERP rendezvous, browser native

Send a file without opening a port.

derphole starts immediately over Tailscale DERP relay, then upgrades the live browser transfer to WebRTC direct when the network allows it. The native CLIs use the same session model with the faster UDP engine.

Native tools

Use the CLI for the fastest path.

The web demo proves the static browser flow. For multi-gigabit paths, long streams, directories, service sharing, and SSH invites, use the native npm packages.

derphole

npx -y derphole@latest send ./photo.jpg
npx -y derphole@latest receive <token>
npx -y derphole@latest ssh invite --user deploy

derpcat

npx -y derpcat@latest listen > disk.img
cat ./disk.img | npx -y derpcat@latest send <token>
npx -y derpcat@latest share 127.0.0.1:3000

What this page is honest about

Browser JavaScript cannot open raw UDP sockets, so this build uses DERP WebSockets for rendezvous and relay fallback, plus WebRTC DataChannel for browser direct paths. The native derpcat and derphole binaries use the same DERP coordination model, but they can also use the optimized direct UDP transport that browsers do not expose.

derpcat and derphole are session-scoped transfer tools. github.com/shayne/derpcat
